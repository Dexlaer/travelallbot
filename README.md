# travel-hunter 🏝️

Личный охотник за дешёвыми поездками в Азию из Сочи/Минвод. Сканирует перелёты
(Aviasales/Travelpayouts) и пакетные туры (Travelata), копит историю цен,
строит карту цен и подсвечивает стоящие варианты. Алерты — в Telegram.

## Как запустить
```bash
python3 hunter.py
```
Сторонние библиотеки не нужны — только Python 3.

## Файлы
- **config.json** — что мониторить и пороги. Это ты правишь сам:
  - `origins` — аэропорты вылета (AER=Сочи, MRV=Минводы)
  - `destinations` — куда смотрим перелёты (IATA-коды городов)
  - `budget.fire` / `budget.ok` — пороги «🔥 огонь» и «потолок» (на двоих, ₽)
  - `max_transfers` — максимум пересадок в одну сторону
  - `tours.countries` — страны для туров (с их ID в Travelata)
- **secrets.json** — токены (Travelpayouts + Telegram). ⚠️ НЕ коммитить, не публиковать.
- **hunter.py** — движок (скан → база → карта → подсветка → Telegram).
- **prices.db** — локальная база истории цен (создаётся сама).

## Как это работает
1. Дёргает API по всем маршрутам/странам: самое дешёвое в рамках `max_transfers`
   + отдельно «комфортное» (максимум 1 пересадка в каждую сторону).
2. Пишет каждую находку в `prices.db` с датой → копится история.
3. Карта цен: цена + медиана по истории + стрелка + сезон (☀️ сейчас / 🌤 скоро / 🌧 не сезон).
4. «🎯 Куда лететь»: ранжирует направления — сезон + поездка целиком
   (перелёт + отель по прикидке `hotel_per_night_two`×`est_nights`, либо тур — что дешевле).
5. Туры: берём только отели с рейтингом ≥ `tours.min_rating` и не ниже 3★.
6. «Стоящее» = поездка целиком ниже порога ИЛИ цена ниже своей медианы на `drop_vs_median_pct`%.

> Медиана начинает работать через несколько дней истории. На холодном старте
> в «стоящее» попадает почти всё, что ниже потолка — это нормально.
